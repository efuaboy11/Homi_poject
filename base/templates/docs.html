API Documentation

Introduction

This is an Order Tracking System where:

  • Store owners upload products
  • Clients (customers) place orders
  • Couriers deliver orders
  • Store owners manage and assign deliveries

The system is divided into four main roles:

  • Admin
  • Store Owner
  • Courier
  • Client

System Roles & Responsibilities

Admin

  • View all user accounts (clients, couriers, store owners)
  • Monitor products and categories
  • Disable or manage user accounts

Store Owner

  • Create and manage products
  • Create product categories
  • Assign couriers to orders
  • Manage delivery process

Courier

  • Receive assigned orders
  • Deliver products to clients
  • Update delivery status

Client

  • Browse products
  • Place orders

Base URL

http://27.102.137.238/api/

Authentication

Login Response

{
  "refresh": "token",
  "access": "token",
  "role": "admin",
  "user_id": "uuid"
}

Field Description

  • refresh – Used to obtain a new access token
  • access – Used for authenticated API requests
  • role – User role (client, courier, store owner, admin)
  • user_id – Unique user identifier (UUID)

Retrieve User Data

Use the following endpoints based on user role:

/couriers/<user_id>/
/clients/<user_id>/
/store-owner/<user_id>/

Important: Always use the correct endpoint for the user role.

User Management

  • /users/ – Get all users
  • /users/<uuid:id>/ – Delete a user

Authentication & Security

  • /request-otp/ – Request OTP
  • /forget-password/ – Reset password
  • /login/ – User login
  • /token/refresh/ – Refresh token

Account Management

  • /disable-account/ – Disable accounts
  • /disable-account/<id>/ – Enable account

Role Registration

  • /couriers/
  • /clients/
  • /store-owner/

Product Categories

  • /product-categories/ – Create & list categories
  • /product-categories/<id>/ – Delete category
  • /delete-multiple-product-categories/

Products

  • /product/ – Create & list products
  • /product/<id>/ – Retrieve/delete product
  • /delete-multiple-product/
  • /product-categories-products/

Cart

  • /cart/
  • /cart/<id>/
  • /increase-cart-product-quantity/
  • /decrease-cart-product-quantity/
  • /remove-cart-product/

Orders

  • /create-order/
  • /order/
  • /order/<id>/
  • /assign-courier/<id>/ – Store owner only
  • /update-order-status/<id>/

Delivery Workflow

  1. Client places order
  2. Store owner processes order
  3. Store owner assigns courier
  4. Courier delivers order
  5. Courier updates delivery status

Note: Couriers do not select orders. They only handle assigned deliveries.

Русская версия

В данной системе клиент оформляет заказ, а владелец магазина управляет процессом доставки.

  • Владелец магазина назначает курьера
  • Курьер выполняет доставку
  • Курьер обновляет статус доставки

Курьеры не принимают заказы самостоятельно. Все назначения выполняются владельцем магазина.

Данная архитектура обеспечивает:

  • Централизованное управление
  • Прозрачность процессов
  • Контроль и ответственность
